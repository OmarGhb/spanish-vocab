## What Paco is

A Spanish vocabulary learning app for French speakers (A2-B1 level). User adds Spanish words; the app generates bilingual definitions, contextual examples, confusable distractors, and lemma metadata via Anthropic. Words enter an FSRS spaced-repetition schedule. Daily review sessions mix fill-in-blank and multiple-choice formats with auto-rated answers the user can override.

The product is mobile-first, French UI, designed for the user (Omar) to actually use daily тАФ not a demo or portfolio piece.

## Architecture decisions worth remembering

### From M2-M3 (still relevant)

- **Spanish-first definitions** (M2.5): definition stored as `{es, fr, pos?}` JSONB. Spanish primary in UI. French behind a "Voir en fran├зais" reveal that counts as a hint in MCQ.
- **Two-step add flow** (M3.1): enrichment is reviewed before saving. `/api/words/enrich` checks the user's deck first; if word exists, returns stored data (no Anthropic call). `/api/words/save` actually inserts. `/api/words/reset-schedule` for due-now duplicates.
- **Wordlist-driven spellcheck over LLM** (M3.2): bundled Spanish dictionary loaded once at module init. No LLM in the spellcheck path.
- **Lemma normalization via Anthropic** (M3.3): the inverse of the M3.2 lesson. When the LLM call is already happening, layering on a separate dataset is overkill. Lemma + form_annotation come back as fields on the existing enrichment response.
- **Form annotation, not duplicate definitions** (M3.3): when a user types "comieron," Anthropic generates the definition for the lemma (*comer*), and a separate `form_annotation` field labels the conjugation (`"Comer тАФ 3┬к pers. plural, pret├йrito perfecto simple"`).

- **Save-route error logging** (M3.3): API routes `console.error` underlying Supabase errors instead of swallowing them under generic 500s.
- **Server-only admin client** (`lib/supabase/admin.ts`): hard `import 'server-only'` guard. Used only by account deletion. Backfill scripts create their own service-role client inline.
- **AbortController plumbing**: navigation away during enrichment cancels the request all the way through to Anthropic SDK.
- **Fire-and-forget event logging** (M3.3 phase B): the five lemma events log via `void fetch().catch(console.error)` тАФ no await, no UI feedback.

- **Wired across the app:** `Button` (a depress `active:scale-[0.98]` added on top of its existing ┬з03 colour/shadow/translate press states; a native disabled `<button>` never receives `:active` so it stays inert without a guard); `WordRow` (scoped to the **non-`flush`, linked** row only тАФ the `flush` variant is wrapped by `SwipeRow`, whose pointer-drag owns the gesture, so a press veil there would flash as the swipe starts); `TopNav` (home/avatar round icons тЖТ `press-icon`/`press-pill-amber` by active state; all nav pills тЖТ `press-pill` / `press-pill-amber`, incl. the locked dict pill which still navigates to the locked screen); Home `HubCard` (`press-card` / `press-card-feature`) + `CollectionSection` header & footer (`press-row`); dictionary entry rows; discover suggestion cards (`press-card`) + topic cards (replaced their ad-hoc `active:bg-tint` with the standard primitive) + close icon; account `ThemePicker` swatches + `components/form` Controls (Stepper, Segmented) + the shared `LinkRow` (account + legal rows); word-detail тЛо trigger + menu items; `AudioButton` circle.
- **Deliberately untouched:** the **locked hub cards and gated controls get NO press class** (non-actionable тАФ the gate is the message); the **review exercise components** (`RatingButtons`, `FillInBlank`, `MultipleChoice`, the session flow) already had feedback (the one surface the guidance names as working), so they were left alone; the swipe Supprimer panel stays owned by its drag gesture.
- **Two honest gaps, logged to `backlog.md` (Mobile UX polish), not silently dropped.** (a) **Hit-target тЙе44px** тАФ the guidance lists it but also forbids layout changes ("ajoute uniquement le feedback d'interaction"); several pills/icons sit at 32тАУ38px, and expanding them is a layout pass, so sizes were left as-is. (b) The `/words` **sort text-toggles** and the ephemeral discover **toast-dismiss ├Ч** were left focus-ring-only тАФ a veil over tiny inline text / a caret reads worse than nothing.
- **Post-device-test tuning (v0.8.12).** The first cut (v0.8.11) shipped the values too quiet to perceive тАФ veil `ink @6%`, `scale .985` тАФ so on a real screen taps looked dead (next to the review buttons' full `bg-tint` swap). Strengthened to veil **10%** (feature **14%**), card depress **`.97`**, small **`.92`**, and added `cursor: pointer` to every `press-*`. `cursor: pointer` fired `:active` on desktop but **not** on iOS Safari, which paints `:active` on tap only when a `touchstart` listener exists тАФ so the no-UI client helper **`TouchActiveFix`** (root layout) adds one empty passive document-level `touchstart` listener, enabling it document-wide. Desktop + iPhone confirmed.
